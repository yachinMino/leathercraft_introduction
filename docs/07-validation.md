# バリデーション

## 方針

- 画面側で操作性向上のための事前チェックを行う
- サーバー側で必須・型・値域・整合性を最終保証する
- マスタ整合性はサーバー側で必ず検証する

## 作品入力バリデーション

| 項目 | 画面側 | サーバー側 |
| --- | --- | --- |
| 作品名 | 必須 | 必須、前後空白除去、1〜100 文字 |
| 革の色 | 選択式 | 必須、マスタ存在確認 |
| シボ | 選択式 | 必須、マスタ存在確認 |
| 糸の色 | 選択式 | 必須、マスタ存在確認 |
| ヘリの処理 | 複数選択 | 選択値ごとにマスタ存在確認、最大 20 件 |
| 鞣し方 | 選択式 | 必須、マスタ存在確認 |
| 出品リンク | 任意 | 空文字または `http/https` の URL、最大 500 文字 |
| 備考 | 任意 | 前後空白除去、最大 4000 文字 |

## 画像バリデーション

| 項目 | 内容 |
| --- | --- |
| 枚数 | 1 作品あたり最大 4 枚 |
| ファイル形式 | `image/*` のみ許可 |
| ファイルサイズ | 1 枚あたり 5MB 以下 |
| 画面側チェック | ファイル選択時に超過を検知してエラー表示 |
| サーバー側チェック | FormData 受信後に再検証 |

### エラーメッセージ例

- `画像は合計 4 枚までです。`
- `画像ファイルのみアップロードできます。`
- `画像は 1 枚あたり 5MB 以下にしてください。`

## 管理画面ログインバリデーション

| 項目 | 内容 |
| --- | --- |
| 必須 | パスワード必須 |
| 文字数 | 1〜50 文字 |
| 認証方式 | サーバー側で設定済みパスワードと比較 |
| 試行回数制限 | 同一 IP あたり 15 分間で 5 回まで |
| 超過時 | 429 エラーで再試行待ち |

### エラーメッセージ例

- `パスワードを入力してください。`
- `パスワードは 50 文字以内で入力してください。`
- `パスワードが間違っています。`
- `ログイン試行回数の上限に達しました。時間をおいて再試行してください。`

## マスタ編集バリデーション

| 項目 | 内容 |
| --- | --- |
| 各カテゴリ | 最低 1 件必要 |
| 項目文字数 | 1〜100 文字 |
| 件数制限 | カテゴリごとに上限あり |
| 重複 | 大文字小文字をならした重複を禁止 |

### カテゴリごとの件数上限

- 革の色: 最大 50 件
- シボ: 最大 20 件
- 糸の色: 最大 20 件
- ヘリの処理: 最大 20 件
- 鞣し方: 最大 20 件

## リアクションバリデーション

| 項目 | 内容 |
| --- | --- |
| 種別 | `like` または `request` のみ |
| 重複送信 | 同一 visitor_id + 同一作品 + 同一種別は重複不可 |
| 認証 | ユーザー登録不要 |

## セキュリティ観点の入力チェック

### SQL インジェクション対策

- D1 への書き込み・検索はプレースホルダ付きクエリを使用
- 作品入力値は文字列として保存
- 選択値はマスタ照合で不正値を拒否

### XSS 対策

- React の標準エスケープで表示
- `dangerouslySetInnerHTML` は未使用
- 作品名・備考はそのまま HTML として描画しない

### 不正リクエスト対策

- 管理 API は Cookie セッション必須
- 管理 API の更新系は同一オリジンのリクエストのみ許可
- 401 時は管理画面へ再ログインを要求
- 管理 API は `Cache-Control: no-store`
- API レスポンスに各種セキュリティヘッダーを付与
- 管理者セッションはサーバー側テーブルで保持し、ログアウト時に無効化
